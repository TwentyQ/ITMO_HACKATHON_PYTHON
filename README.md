# Каталог книг "НаПолку!" - команда "Омори"
## Веб-приложение для управления коллекцией книг с общей базой для всех пользователей.

Ссылка на видео защиты: https://disk.yandex.ru/i/Cp13Cp1z_pphhw

## **Установка:**

1. Клонируйте репозиторий.

2. Создайте виртуальное окружение:
python -m venv venv
source venv/bin/activate для Linux/Mac
или
venv\Scripts\activate для Windows

3. Установите зависимости:
pip install django pillow

4. Запустите сервер:
python manage.py runserver

5. Откройте в браузере: http://127.0.0.1:8000/


## **Модели**

**Book (Книга)**
Содержит информацию о литературном произведении: название, автора, год издания, жанр, краткое описание и обложку. Каждая книга существует в единственном экземпляре в общем каталоге.

**UserStatus (Статус пользователя)**
Связывает пользователей с книгами через систему статусов чтения. Каждый пользователь может установить для каждой книги один из пяти статусов: "Не начата", "Читаю", "Прочитана", "В планах" или "Брошена".

**User (Пользователь)**
Стандартная модель Django для управления учетными записями пользователей, расширенная связями с книгами через UserStatus.

## **Контроллеры**

Основные контроллеры:

* home() - обрабатывает главную страницу со статистикой и списком пользователей

* book_list() - формирует каталог всех книг в системе

* book_create() - управляет процессом добавления новых книг

* book_detail() - предоставляет детальную информацию о конкретной книге

* book_edit() - отвечает за редактирование существующих книг

* login_view(), register_view() и logout_view() - реализуют систему аутентификации

* my_books() - формирует персональную страницу пользователя с группировкой книг по статусам
  
## **Шаблоны**

Ключевые шаблоны:

* main_page.html - главная страница с общей статистикой и таблицей пользователей

* catalog.html - визуализация книжного каталога в виде полок с книгами

* forms.html - универсальная форма для создания и редактирования книг

* book.html - детальная карточка книги с управлением статусами

* login.html и register.html - страницы аутентификации в едином стиле

* my_books.html - персональный кабинет пользователя с группировкой книг по статусам чтения

## **Особенности реализации:**

* Единая дизайн-система с фирменной цветовой схемой (розовый,черный и белый)

* Адаптивная вёрстка, корректно отображающаяся на всех устройствах

* Интерактивные элементы с анимациями (поднятие книг при наведении)

## **Маршрутизация**

/ - корневой путь к главной странице

/catalog/ - отображение всего каталога книг

/catalog/new/ - форма добавления новой книги

/catalog/book/?id=X - детальная информация о книге с идентификатором X

/catalog/edit/?id=X - форма редактирования существующей книги

/profil/ - личный кабинет пользователя

/login/ и /register/ - страницы аутентификации

/logout/ - выход из системы

## **Работа с базой данных**
Приложение использует SQLite с Django ORM для управления данными:

* Автоматическое создание схемы базы данных через миграции

* Доступ к данным без написания SQL-запросов


# **Организация работы в команде**
## **Методология GitHub Flow**

Ветвление:

**main** - основная ветка, всегда содержит рабочую стабильную версию

**feature/frontend-templates** - ветка для разработки HTML-шаблонов и интерфейса

**feature/add-database-models** - ветка для создания и настройки моделей данных

**feature/add-django-backend** - ветка для разработки серверной логики и контроллеров

**fixes** — ветка для исправления ошибок и багов

1) Создание новой feature-ветки от main для каждой значимой задачи

2) Регулярные коммиты с осмысленными сообщениями о изменениях

3) Push ветки в удаленный репозиторий для синхронизации

5) Проведение Code Review коллегой по команде
